How it works

Three steps from text file to AI hallucination.

01

Plant

An attacker adds false information to robots.txt as comments or fake Disallow paths. Or they craft a malicious llms.txt — a file literally designed to instruct LLMs.

02

Crawl

AI crawlers (GPTBot, ClaudeBot, PerplexityBot, Google-Extended) ingest robots.txt and llms.txt as part of their web crawl. The content enters the model's context.

03

Poison

When users ask an AI about the site, the model may regurgitate the injected false data as if it were factual. The prompt injection becomes the AI's "knowledge."

Attack scenarios

Real-world scenarios where robots.txt and llms.txt poisoning is dangerous.

Reputation manipulation

Inject false CEO names, revenue figures, or company history. AI assistants present the fabricated data as fact when users ask about the organization.

Phishing via AI

Inject fake support email addresses or phone numbers. When users ask an AI "how do I contact [company]?", the model directs them straight to the attacker.

Competitor sabotage

If an attacker compromises a competitor's web server, they can poison robots.txt to spread false information through every AI-powered search engine simultaneously.

SEO / AI poisoning

Manipulate what AI-powered search engines (Perplexity, Google AI Overviews, Bing Copilot) say about any domain. Traditional SEO poisoning, evolved.

Trust exploitation via llms.txt

The llms.txt standard is literally an instruction file for LLMs. A compromised llms.txt is game over — it's designed to be trusted and followed by AI systems.

Why this matters

This isn't a theoretical attack. The infrastructure for it already exists on every website.

  • robots.txt exists on virtually every website. It's one of the oldest conventions on the web.
  • AI crawlers read it by design. GPTBot, ClaudeBot, PerplexityBot, Google-Extended all parse robots.txt.
  • No validation, no signatures, no integrity checks. There is no way to verify the authenticity of robots.txt content.
  • The llms.txt standard makes it worse. It's literally an instruction file for LLMs — designed to be ingested and followed.
  • Comments are read by AI. Even though # lines are "comments" in robots.txt, LLM crawlers process the full file content.
  • There is no defense. You can't stop AI from reading your robots.txt. It's a public file by definition.
